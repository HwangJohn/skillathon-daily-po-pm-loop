# Codex Community Korea Guide Index

Source: https://codex-community-korea.lovable.app/learn

This file summarizes the guide set used for the Skillthon build. It is not a full copy of the site. It keeps only the parts needed to build and present the Daily PO/PM Loop prototype.

## Most relevant guides

| Guide | Why it matters |
| --- | --- |
| `g-002` Skillthon Onboarding | Frames the event as reusable workflow creation, not one-off prompting. |
| `skillthon-submission-guide` | Defines the expected submission shape: problem, mock data, prompt, SKILL.md, validation, limits. |
| `event-lab-skillthon-template` | Useful as the submission story template. |
| `skill-what-is-skill` | Explains the reusable procedure concept. |
| `skill-skill-md` | Guides SKILL.md structure and trigger clarity. |
| `skill-good` | Helps evaluate whether the skill is reusable and specific. |
| `skill-reusable` | Reinforces mock-data-first and repeatable I/O. |
| `skill-quality` / `skill-evals-basic` | Supports validation and demo scoring. |
| `harness-agents-vs-skill` | Clarifies why this belongs in SKILL.md rather than repo instructions. |
| `harness-validation-loop` | Provides the before/after verification loop. |
| `advanced-computer-use-qa` | Supports the demo idea where Codex inspects the generated UI. |

## Track mapping

- Skillthon requirements: `g-002`, `skillthon-submission-guide`, `event-lab-skillthon-template`
- Skill authoring: `skill-what-is-skill`, `skill-skill-md`, `skill-good`, `skill-reusable`
- Harness and validation: `harness-*`, `advanced-computer-use-qa`
- Expansion: `advanced-hooks`, `advanced-mcp`, `advanced-plugins`, `advanced-subagents`, `advanced-long-work-splitting`

## Build implication

The prototype should present itself as a Codex Skill submission first. The local app is a visual demo surface for the skill's workflow, not the main artifact.
