# Daily PO/PM Loop Concept

## Problem

Daily briefings collect information, but PO/PMs stop reading them when they do not help with decisions. Outlook, Jira, calendar, and todos are already visible elsewhere. The missing value is the decision loop.

## Product idea

Daily PO/PM Loop turns scattered signals into a guided HITL workflow:

1. Triage daily signals.
2. Commit to a small focus set.
3. Identify product opportunity candidates.
4. Let the PO/PM edit and add opportunity cards.
5. Generate PRD and UI mock only after the user selects a card.
6. Produce pet prompts and next automation prompts.

## Demo wow moment

The demo should not immediately jump from signals to generated design. The stronger HITL moment is:

1. Codex proposes opportunity cards.
2. User edits a card.
3. User clicks refine to polish the card without losing intent.
4. User clicks generate plan.
5. Prototype shows a PRD and UI mock derived from that card.

## Positioning

This is not a todo app. It is a Codex-native PO/PM operating loop that converts daily signals into decisions and product artifacts.

## Demo line

> "Codex does not replace the PO/PM. It keeps the decision loop tight, asks the right questions, and turns one approved opportunity into a planning mock."
