# Codex Harness and Automation Notes

## Codex-native surfaces

This prototype should actively use Codex/OpenAI surfaces where they fit:

- Codex Skill: stores the reusable PO/PM workflow.
- Codex Pet: ambient cue surface for short active-thread prompts.
- Codex Automations or batch-style scheduling: future path for morning, midday, and end-of-day runs.
- Computer Use / in-app browser: future QA path for inspecting the generated mock screen.
- MCP / plugins: future path for Outlook, Jira, Slack, Calendar, and GitHub Issues.

## Current MVP behavior

- Pet is represented through `pet_prompt` fields generated by the skill and displayed in the prototype.
- Automation is represented through `next_automation_prompt` fields.
- External connectors are represented through mock signal data.

## Why this is acceptable

The Skillthon submission tests repeatability and workflow design. Real integrations would make the demo brittle and distract from the core skill.

## Expansion path

1. Replace mock data with MCP/CLI adapters.
2. Schedule the three daily skill prompts.
3. Route short progress messages into Codex Pet.
4. Use Computer Use to inspect the local board and generated mock.
5. Add eval prompts for common PO/PM scenarios.
